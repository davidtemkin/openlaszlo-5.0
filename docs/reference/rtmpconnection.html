<rtmpconnection>

Allows connections to RTMP servers

Extends LzView

rtmpconnection allows applications to connect to RTMP servers, like http://osflash.org/red5 or http://www.wowzamedia.com/. Note that you'll need a server running with the appropriate URLs set up for this example to work. 


                <canvas>
                    <rtmpconnection src="rtmp://localhost:1935/simplevideostreaming/" autoconnect="true"/>
                    <videoplayer url="Extremists.flv" type="rtmp" autoplay="true" width="320" height="240"/>
                </canvas>

Attributes

NameTypeDefaultDescription
altsrcstringAlternate application url, used if the connection for src fails for example : "rtmp:/appname" or "rtmp://myserver.com/myapp/".
autoconnectbooleantrueConnect automatically during init. If false, you need to explicitly call connect().
connectionparamsexpressionnullConnection parameters that will be sent to the server when NetConnection.connect() is called
connectiontimeoutnumber3000Number of milliseconds to wait for connection at src before using altsrc URL
debugexpressionfalseTurn on for verbose debugger output.
objectencodingnumber0The AMF object encoding to use for this connection, either 0 or 3.
srcstringApplication url, for example "rtmp:/appname" or "rtmp://myserver.com/myapp/".
stagenumber0Number to indicate what stage of connection we're in: 0: disconnected, 1: connecting, 2: connected.
statusstringdisconnectedString to indicate connection status.
