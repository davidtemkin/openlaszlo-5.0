<mediastream>

Allows video playback, live broadcast or recording .

Extends node

mediastream allows applications to stream video from HTTP or RTMP servers. Classes like <videoview> use mediastreams to handle the connection to the server. 


        <canvas>
            <mediastream name="ms" autoplay="true" type="http" url="http://www.archive.org/download//JudgeMediaTestVideoFile_0/video.flv"/>
            <videoview type="http" autoplay="true" width="320" height="240" stream="canvas.ms"/>
        </canvas>

rtmpexpressionnullThe rtmp connection to use (defaults to the first one created).

timenumber0Current time of mediastream currently playing or recording (in seconds)
totaltimenumber0Total length (in seconds) of mediastream currently playing.
typestringhttpProtocol "rtmp" or "http".
urlstringThe url of the mediastream, maybe be relative URL, for example: http://localhost/myvideo.flv, or simply myvideo.flv when type="rtmp" the url is always relative to the rtmpconnection.
