<videoview>

Allows video playback.

Extends view

videoview allows applications to play video from HTTP or RTMP servers. 


            <canvas>
            <videoview url="http://www.archive.org/download//JudgeMediaTestVideoFile_0/video.flv" type="http" autoplay="true" width="320" height="240"/>
            </canvas>
✎︎ Edit

Attributes

NameTypeDefaultDescription
autoplayexpressionfalseIf true, video will start playing as soon as url is set.
buffertimeexpression0If true, video will start playing as soon as url is set.
camexpressionnullThe camera object.
debugexpressionfalseIf true, Debug.write extra stuff (this may go away).
micexpressionnullThe microphone object.
playvolumeexpression1.0The volume for video playback, a float from 0 to 1.
starttimenumber0Show the frame from this time (in seconds) whenever the url is set for video playback. Use -1 to show no frame (which will leave the image as it was when switching videos)
streamexpressionfalseThe mediastream object. A mediastream will be automatically generated if one is not specified at init time.
typestringhttpProtocol "rtmp" or "http".
urlstringThe url of the video, may be a relative URL, for example: "http://localhost/myvideo.flv", or simply "myvideo.flv". When type="rtmp", the url is always relative the rtmpconnection. (Use only when you are not explicitly declaring the mediastream.)
